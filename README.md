# Console File Manager

Консольный файловый менеджер на C#/.NET. Проект сделан как учебная работа по ООП: основной акцент здесь не на количестве функций, а на том, как эти функции разделены по слоям и расширяются без переписывания существующей логики.

Приложение позволяет подключиться к локальной файловой системе, перемещаться по каталогам, выводить дерево директорий, читать файлы и выполнять базовые операции над файлами через текстовые команды.

## Что умеет приложение

- подключение к локальной файловой системе;
- отключение от текущей файловой системы;
- переход по каталогам через относительные и абсолютные пути;
- поддержка `.` и `..`;
- вывод дерева каталогов с настраиваемой глубиной;
- просмотр содержимого файла;
- копирование, перемещение, удаление и переименование файлов;
- обработка некорректных команд и ошибок выполнения;
- работа в интерактивном консольном режиме.

## Примеры команд

```text
connect C:\Users\User\Documents -m local
tree list -d 2
tree goto projects
tree goto ..
file show notes.txt -m console
file copy notes.txt backup
file move notes.txt archive
file rename notes.txt README.txt
file delete README.txt
disconnect
q
```

`q` завершает работу приложения.

## Архитектурная идея

Код разделён на две основные части:

```text
src/
├── FileManager.Core/          # доменная логика, команды, работа с файловой системой
└── FileManager.Presentation/  # консольный интерфейс и парсинг команд

tests/
└── FileManager.Tests/         # тесты парсера команд
```

`Core` не зависит от консоли и конкретного способа ввода команд. В нём находятся команды, состояние подключения, абстракция файловой системы, результаты выполнения и логика построения дерева.

`Presentation` отвечает за консольный цикл и превращение введённой строки в команду. Это позволяет менять способ ввода команд, не трогая основную логику.

## Что в проекте было важно спроектировать

### Команды

Каждое действие представлено отдельной командой:

- `ConnectCommand`;
- `DisconnectCommand`;
- `TreeGotoCommand`;
- `TreeListCommand`;
- `FileShowCommand`;
- `FileCopyCommand`;
- `FileMoveCommand`;
- `FileDeleteCommand`;
- `FileRenameCommand`.

У команд единый контракт выполнения. Благодаря этому консольному интерфейсу не нужно знать детали конкретной операции: он получает команду от парсера и запускает её.

### Парсинг

Парсер построен как цепочка обработчиков. Отдельные звенья отвечают за разные команды, подкоманды, позиционные аргументы и флаги.

Такой подход делает добавление новой команды или нового флага более локальным: не нужно превращать парсер в один большой `switch` или набор вложенных `if`.

### Состояние подключения

Контекст приложения может быть в двух состояниях:

- подключён к файловой системе;
- не подключён.

Это ограничивает выполнение команд: например, операции с файлами не должны работать до `connect`.

### Абстракция файловой системы

Работа с файлами идёт через `IFileSystem`. Сейчас реализована локальная файловая система, но основная логика команд не привязана напрямую к `System.IO`.

Это оставляет возможность добавить другой источник данных: например, in-memory файловую систему для тестов или удалённое хранилище.

### Отрисовка дерева

Вывод дерева отделён от самой файловой системы. Для дерева используются отдельные компоненты и visitor, а символы для файлов, директорий и отступов вынесены в отдельную настройку.

Обход директорий выполняется через `IEnumerable`, поэтому структура файловой системы не загружается в память целиком заранее.

## Использованные идеи и паттерны

В проекте применяются:

- **Command** — отдельный объект под каждое действие пользователя;
- **Chain of Responsibility** — цепочки парсеров команд, аргументов и флагов;
- **Builder** — постепенная сборка команд из аргументов;
- **State** — разные правила поведения для подключённого и отключённого состояния;
- **Composite / Visitor** — построение и обход элементов дерева файловой системы;
- **Factory** — создание парсеров и объектов для отрисовки;
- **Dependency Inversion** — команды зависят от абстракций, а не от конкретной консоли или конкретной файловой системы.

## Запуск

Склонировать репозиторий:

```bash
git clone <repository-url>
cd <repository-folder>
```

Восстановить зависимости:

```bash
dotnet restore
```

Собрать проект:

```bash
dotnet build
```

Запустить тесты:

```bash
dotnet test
```

Запустить консольное приложение:

```bash
dotnet run --project src/FileManager.Presentation/FileManager.Presentation.csproj
```

## Тесты

Сейчас тестами покрыт парсер команд: проверяется, что корректный ввод создаёт команду нужного типа, а неполные или лишние аргументы приводят к ошибке парсинга.

Примеры проверяемых сценариев:

- `connect` с адресом и режимом;
- `disconnect` без лишних аргументов;
- `tree goto` и `tree list -d`;
- `file show`, `file copy`, `file move`, `file delete`, `file rename`;
- ошибки при пропущенных параметрах и лишних токенах.
